$DCPS.ON#

Description: Enable the DCPS
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: 1
Example: $DCPS.ON 1

Enables the DCPS, $DCPS.ON 1 to turn it on, and $DCPS.ON 0 to turn it off. This can be useful to conserve battery.

$DCPS.RP#

Description: Run order
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: 9
Example: $DCPS.RP 1

Sets the Run order for the sensor. See Run order

$DCPS.RC#

Description: Run counter
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: -
Example: $DCPS.RC 1

Sets the Run counter for the sensor. See Run counter

$DCPS.GPS#

Description: Enable the GPS
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: 2
Example: $DCPS.GPS 1

To compensate for vehicle movement the GPS must be enabled. DCPS.GPS = 1: The GPS is left powered on when the DCPS is acquiring data (faster fix). DCPS.GPS = 2: The GPS is powered off when the DCPS is acquiring data(saves power)

$DCPS.DEV#

Description: Set the magnetic declination
No. of parameters: 1
Unit: degrees
Parameter type: integer
Minimum parameter value: -180
Maximum parameter value: 180
Example: $DCPS.DEV -21

The magnetic deviation used when transmitting surface currents. This will need to be set depending on location, Baltic Sea is ~+4°, while the SO can be ~-20°. Find out more about magnetic declination here

$DCPS.MIN#

Description: Measurement time
No. of parameters: 1
Unit: minutes
Parameter type: integer
Minimum parameter value: 2
Maximum parameter value: -
Example: $DCPS.MIN 4

The number of minutes the DCPS is set to acquire data. This setting should be set on 2 min intervals. 2,4,6,8 minutes etc., depending on the DCPS configuration.

$DCPS.AVG#

Description: Cell averaging
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: -
Example: $DCPS.AVG 6

The number of cells down from surface used to calculate the average current.

$DCPS.LOG#

Description: Log output
No. of parameters: 1
Unit: -
Parameter type: integer
Minimum parameter value: 0
Maximum parameter value: 1
Example: $DCPS.LOG 1

Log sensor output to disk.

Data fields#

In the Datalogger data tab the following columns are displayed:

Status indicator bits:

0: No errors
1: Memory error
2: File Open error
4: Timeout error
8: GPS error

Note

The binary sytem makes it possible for the sensor to communicate more than one error. E.g., if we see 9, we know that there was a memory and GPS error.

The DCPS is preconfigured when delivered, and if you are reconfiguring the DCPS, make sure the output from the DCPS contain at least these values:

  • Cell Index

  • Horizontal Speed[cm/s]

  • Direction[Deg.M]

This is so that the datalogger will be able to process the DCPS data. All other values will be ignored, but saved in the DCPS.txt.

To make changes to the internal sensor settings, make sure to follow the step outlined in the , until you get to the Direct serial. After you have closed the terminal, you can use the manufacturers own software to make changes. This described more under DCPS configuration.

Logged output from DCPS#

The following output is saved to “DCPS.TXT” file on disk

  • Black text: This text is generated by the datalogger and contains date/time of the internal datalogger clock when saving to disk

  • Green text: This is the direct raw output of the DSCP instrument. To process this data, see DCPS manuals.

  • Blue text: This is the calculated current speed and direction calculated by the datalogger based on the raw DCPS output. Units are m/s and degrees.

  • Red text: GPS position and time of when the DCPS started and stopped acquisition.

Algorithm#

The following algorithm is used to process the data:

  • Read direction and speed for each cell from the DCPS in the “MEASUREMENT” line

  • Converts direction and speed to northing and easting speed.

  • Calculatean average northing and easting speed for each cell if multiple records are received.

  • Subtracts (actually add) the northing and easting speed derived from the GPS position at start and end of the acquisition, from each cell.

  • Calculate the average northing and easting speed from the first N cells (decided by $DCPS.AVG).

  • Convert the average to direction and speed (this is the surface current).
